`Registry` 是 Huoban 的字架：它声明对象来源和索引范围。当前 v1alpha1 定义 Registry 对象形状，但 source repo 没有 Registry loader、远程 discovery 或 hosted registry runtime。

设计背景见 [Registry 模型](/registry-model)。

## 字段

| 字段 | 要求 | Schema 语义 |
| --- | --- | --- |
| `spec.sources` | 必填，至少 1 项 | source 数组。 |
| `spec.sources[].type` | 必填 | `local`、`git`、`http` 或 `mcp`。 |
| `spec.sources[].path` | 与 `url` 至少存在一个 | 字符串。 |
| `spec.sources[].url` | 与 `path` 至少存在一个 | 字符串。 |
| `spec.index` | 可选 | 只允许 `includeKinds`。 |
| `spec.index.includeKinds` | 可选 | 字符串数组。 |
| `spec.trustPolicyRef` | 可选 | local object reference。 |
| `status` | 可选 | 通用状态。 |

source 使用 `anyOf`，因此 path 与 url 可以同时存在。schema 不按 `type` 约束 locator：例如它不强制 `local` 使用 path，也不强制 `http` 使用 url。

`includeKinds` 没有 kind 枚举、`minItems` 或 `uniqueItems`；它是当前 schema 中的字符串过滤列表，不是权限规则。

## 合法示例

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Registry
metadata:
  name: local-huoban-registry
spec:
  sources:
    - type: local
      path: ./examples
  index:
    includeKinds:
      - Skill
      - Adapter
      - Flow
      - Profile
      - Policy
      - Run
      - Checkpoint
      - Artifact
      - Trust
```

对应 source example：`examples/registry/local-registry.yaml`。

## 当前 CLI 行为

- `validate` 检查 source、枚举、locator 和 index 字段形状。
- `explain` 输出 source 数量、type/locator 和 `includeKinds`。
- workspace explain 的对象索引来自命令行已读取的文件，不会读取 Registry sources。
- 当前 Preflight 不消费 Registry，不解析 `trustPolicyRef`，也不通过 Registry 发现缺失对象。

## 验证边界

- schema 不检查 path 是否存在或 URL 是否可访问。
- schema 不把 `includeKinds` 限制为 10 个顶层 kind。
- `trustPolicyRef` 只验证引用形状；当前 CLI 不解析或应用它。
- Registry 不内联对象内容，也不表达来源已被信任。
- 当前 source 没有 registry 发布、版本选择、缓存或推荐协议实现。

## 相邻概念边界

| 容易混淆 | 精确边界 |
| --- | --- |
| Registry / workspace | Registry 声明来源；当前 CLI workspace 是命令行实际读入的对象集合。 |
| Registry / Trust | Registry 说明去哪里找；Trust 记录来源和审查。 |
| Registry / Adapter | Registry 声明对象来源集合；Adapter 标准化一个导入格式或外部能力。 |
| `includeKinds` / Policy | 前者是索引范围字符串；后者声明权限规则。 |
| Registry / marketplace | 当前对象只定义来源与索引字段，不定义市场、排名或分发。 |

## 常见错误

- 声称写了 Registry 就会自动加载 source。
- 把 `includeKinds` 当作执行 allowlist。
- 假设 source type 与 path/url 组合已被 schema 强制。
- 认为 `trustPolicyRef` 可解析或已经应用。
- 在 Registry 中内联 Skill、Flow 等对象正文。
- 把 Requirement、Preflight Report 或 Lifecycle Handler 加入顶层 kind 列表。
