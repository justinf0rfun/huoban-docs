本页给出 Huoban 仓库的最短验证路径。完成后，你应该确认三件事：标准对象能通过 schema 校验，CLI 能解释 workspace，Flow 能生成计划态 dry-run `Run`。

本页验证的是标准对象和 CLI 解释能力，不会触发真实 agent 执行。

## 前置条件

使用 Node.js 20 或更高版本。

```bash
node --version
npm --version
```

所有命令都在 Huoban 仓库根目录运行。

<Steps>
  <Step title="安装依赖">

```bash
npm install
```

  </Step>

  <Step title="运行完整检查">

```bash
npm test
```

`npm test` 会导入 Profile、派生 Policy、导出 AGENTS.md、生成 dry-run Run、验证 YAML 对象、解释 workspace，并检查生成文件是否漂移。

  </Step>

  <Step title="验证标准对象">

```bash
npm run validate
```

验证单个 Flow：

```bash
npm exec -- huoban validate examples/flows/idea-to-spec-review.yaml
```

预期输出：

```text
✓ examples/flows/idea-to-spec-review.yaml: Flow/idea-to-spec-review
```

  </Step>

  <Step title="解释 workspace">

```bash
npm run explain
```

预期输出开头：

```text
Huoban Workspace
  objects: 15
  kinds: Adapter=1, Artifact=1, Checkpoint=1, Flow=1, Policy=2, Profile=3, Registry=1, Run=2, Skill=2, Trust=1
  print process:
```

如果只想解释核心 Flow：

```bash
npm run explain:flow
```

  </Step>

  <Step title="验证 Profile 导入导出">

这一步验证 Huoban 可以把已有上下文文件转成结构化 `Profile`，并导出回 agent 可读取的文件。

```bash
npm run import:profile
npm run export:agents
npm run derive:policy
```

  </Step>

  <Step title="生成 dry-run Run">

```bash
npm run print:dry-run
```

该命令基于 `examples/flows/idea-to-spec-review.yaml` 生成计划态 `Run`，不会执行 agent 工作。

预期输出：

```text
✓ examples/flows/idea-to-spec-review.yaml -> examples/generated/idea-to-spec-review-dry-run.yaml: Run/idea-to-spec-review-dry-run
```

  </Step>
</Steps>

## 本页验证了什么

| 文件 | 对象 | 被验证的职责 |
| --- | --- | --- |
| `examples/flows/idea-to-spec-review.yaml` | `Flow` | 声明 capability-first 的编排、阶段、checkpoint 和 artifact。 |
| `examples/profiles/huoban-open-source-project.yaml` | `Profile` | 注入项目上下文、设计哲学和文档写作规则。 |
| `examples/policies/safe-defaults.yaml` | `Policy` | 为高风险 side effects 设置审批或拒绝规则。 |
| `examples/adapters/grill-me-skill-md.yaml` | `Adapter` | 把 `SKILL.md` 来源标准化成 Huoban 能力。 |
| `examples/generated/idea-to-spec-review-dry-run.yaml` | `Run` | 记录 Flow、Profile、Policy 和 binding 的计划态快照。 |
| `examples/checkpoints/idea-to-spec-review-direction.yaml` | `Checkpoint` | 在方向确认处建立人类决策边界。 |
| `examples/artifacts/review-notes.yaml` | `Artifact` | 记录 run 产出的 review notes。 |
