# Policy 对象

`Policy` 是 Huoban 的印坊规矩：它决定什么允许、什么禁止、什么必须经过审批。

Policy 必须与 Profile 分离。Profile 影响判断，Policy 控制行为边界。

## 规则中心

Policy 不应主要按具体命令匹配，例如 `git push` 或 `rm -rf`。核心规则应围绕：

- capability
- side effect
- trust level

当前 v1alpha1 的 `rules[].when` 只标准化 `capabilities`、`sideEffects` 和 `trustLevels`。runtime mode、artifact type 和 checkpoint requirement 可以作为未来扩展或实现侧语义，不能写成当前 schema 已支持的匹配字段。

## spec 字段

| 字段 | 要求 | 说明 |
| --- | --- | --- |
| `rules` | 必填 | 策略规则数组。 |
| `rules[].effect` | 必填 | `allow`、`deny`、`requireApproval`。 |
| `rules[].when` | 必填 | 匹配条件。 |
| `defaults` | 可选 | 未声明副作用的默认 effect。 |

## side effects

标准 side effects 包括：

- `readFile`
- `writeFile`
- `deleteFile`
- `executeCommand`
- `modifyGitIndex`
- `modifyGitHistory`
- `networkAccess`
- `sendMessage`
- `publishRemote`
- `spendMoney`
- `accessSecret`
- `installDependency`
- `changePermission`

## 最小示例

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Policy
metadata:
  name: safe-defaults
spec:
  defaults:
    unknownSideEffect: requireApproval
  rules:
    - effect: requireApproval
      when:
        sideEffects:
          - publishRemote
          - spendMoney
          - accessSecret
    - effect: deny
      when:
        sideEffects:
          - modifyGitHistory
```

## status

常见 conditions：

- `PolicyValidated`
- `PolicyDerived`
- `PolicyNeedsReview`

## 设计约束

declared side effects（`sideEffects.declared`）是声明，不等于事实。runtime 还必须在 `Run.status.observedSideEffects` 记录实际观察到的副作用，并在超出声明时触发 Run condition。
