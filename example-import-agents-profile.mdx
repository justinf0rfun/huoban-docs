这个 public example 使用当前 CLI 支持的迁移路径：

```text
AGENTS.md -> Profile -> review -> derived Policy
                    -> exported AGENTS.md -> review
```

`Profile` 保存分层上下文；`Policy` 保存副作用、审批和权限边界。导入与派生都会标记 review 状态，但当前 CLI 没有“接受推断”的命令，也不会在 review 后自动迁移 status。

文中的 `examples/...` 是 Huoban source repo 内的路径，不是文档站链接。

## 复现命令

在 source repo 根目录运行：

```bash
npm install
npm run import:profile
npm run derive:policy
npm run export:agents
npm run validate
npm run explain
```

对应的底层命令逐字来自 `package.json`：

```bash
node ./bin/huoban.js import examples/import/AGENTS.md --as Profile --out examples/generated/agents-profile.yaml
node ./bin/huoban.js derive policy examples/generated/agents-profile.yaml --out examples/derived/agents-risk-policy.yaml
node ./bin/huoban.js export examples/generated/agents-profile.yaml --to AGENTS.md --out examples/export/AGENTS.md
```

当前 import 只支持 `--as Profile`；当前 export 只支持 `--to AGENTS.md`，并且要求显式 `--out`。

## 完整输入

下面逐字取自 `examples/import/AGENTS.md`：

```md
# Agent Rules

## Style

Prefer small surgical changes. Do not refactor unrelated code.
Use test-first changes for bug fixes when practical.

## Commands

Run `npm run validate` before finalizing schema or example changes.
Use `npm run explain` to inspect the example workspace.

## Risk

Ask before modifying git history, publishing remote changes, or accessing secrets.
```

## 完整生成 Profile

下面逐字取自 `examples/generated/agents-profile.yaml`：

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Profile
metadata:
  name: agents-profile
  labels:
    huoban.dev/imported-from: examples/import/AGENTS.md
spec:
  layers:
    - name: style
      data:
        preferSmallChanges: true
        avoidUnrelatedRefactor: true
        testFirstForBugfixes: true
      content: |-
        Prefer small surgical changes. Do not refactor unrelated code.
        Use test-first changes for bug fixes when practical.
    - name: commands
      data:
        commands:
          - npm run validate
          - npm run explain
      content: |-
        Run `npm run validate` before finalizing schema or example changes.
        Use `npm run explain` to inspect the example workspace.
    - name: risk
      data:
        requiresApproval:
          - publishRemote
          - modifyGitHistory
          - accessSecret
        inferred: true
      content: Ask before modifying git history, publishing remote changes, or accessing secrets.
status:
  conditions:
    - type: ProfileImported
      status: "True"
      reason: ImportedFromMarkdown
      message: examples/import/AGENTS.md was converted into Profile layers.
    - type: ImportNeedsReview
      status: "True"
      reason: RuleBasedInference
      message: Structured fields were inferred by deterministic rules and should be reviewed.
```

导入器保留 `content`，同时按当前规则抽取 `style`、`commands` 和 `risk` data。`ImportNeedsReview=True` 表示这些结构化字段是规则推断，不是可信度或授权结论。

## 完整派生 Policy

下面逐字取自 `examples/derived/agents-risk-policy.yaml`：

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Policy
metadata:
  name: agents-profile-risk-policy
  labels:
    huoban.dev/derived-from: Profile/agents-profile
spec:
  defaults:
    unknownSideEffect: requireApproval
  rules:
    - effect: requireApproval
      when:
        sideEffects:
          - publishRemote
          - modifyGitHistory
          - accessSecret
status:
  conditions:
    - type: PolicyDerived
      status: "True"
      reason: DerivedFromProfileRiskLayer
      message: Derived from Profile/agents-profile risk layer.
    - type: PolicyNeedsReview
      status: "True"
      reason: RuleBasedDerivation
      message: Policy was derived by deterministic rules and should be reviewed before enforcement.
```

`derive policy` 读取名为 `risk` 的 layer，并要求 `data.requiresApproval` 是非空数组。它不会判断规则是否符合团队意图，也不会执行 Policy。

## 导出结果

`npm run export:agents` 生成 `examples/export/AGENTS.md`。文件开头明确要求 review：

```md
# Agent Rules

<!-- Generated from Huoban Profile. Review before use. -->

Source Profile: `agents-profile`
```

导出 Markdown 是工具消费格式，不应因为“由 Huoban 生成”就被当作可信协议对象。需要 review 后再交给 agent 工具使用。

## 审查清单

| 检查项 | 要确认的事实 |
| --- | --- |
| 原文保留 | 每个 layer 的 `content` 没有丢失关键限定语。 |
| data 推断 | 布尔值、命令和 side effect 名称符合原意。 |
| 风险边界 | 审批规则进入独立 Policy，而不是只停留在 Profile 文本。 |
| 未知副作用 | `unknownSideEffect: requireApproval` 是否符合项目要求。 |
| 输出用途 | 导出的 AGENTS.md 仍经过人工或既有治理流程。 |

当前 CLI 不提供 review workflow。不要虚构“点击接受后自动把 `ImportNeedsReview` 变为 False”之类的状态迁移；review 结论应由采用方的流程记录。

## 与 Run snapshot 的关系

如果 Flow 或 Run 引用该 Profile，当前 `print --dry-run` 使用 `profileRefHash` 哈希 `profileRefs` 数组。它不哈希 effective Profile 内容。

```yaml
snapshot:
  profileRefHash: sha256:...
```

Preflight 的 `snapshotHash` 则哈希整个 `Run.spec`。两者都不能单独证明 Profile 正文已被冻结或获准使用。

## 结果边界

| 成功结果 | 可以确认 | 不能据此确认 |
| --- | --- | --- |
| `ProfileImported=True` | Markdown 已转换为 schema-valid Profile 形状。 | 推断正确或内容可信。 |
| `PolicyDerived=True` | risk data 已按确定规则转成 Policy。 | Policy 已审查或已执行。 |
| export 完成 | Profile 已呈现为 AGENTS.md。 | 下游 agent 应无条件信任该文件。 |
