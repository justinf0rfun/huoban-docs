`Checkpoint` 是 Huoban 的校对对象：它把一次具体 Run 中的人类或系统决策边界记录为独立资源。Flow 可以声明 checkpoint 位置；Checkpoint 对象记录实例。

设计背景见 [Checkpoint 模型](/checkpoint-model)。

## 字段

| 字段 | 要求 | Schema 语义 |
| --- | --- | --- |
| `spec.runRef` | 必填 | local object reference。 |
| `spec.stage` | 可选 | stage id 字符串。 |
| `spec.reason` | 必填 | 原因字符串。 |
| `spec.requiredActions` | 可选 | 不重复的动作数组；可以为空。 |
| `spec.artifactRefs` | 可选 | Artifact 引用数组。 |
| `spec.expiresAt` | 可选 | 字符串；schema 不校验日期时间格式。 |
| `status` | 可选 | 通用状态；`phase` 是未限定枚举的字符串。 |

`requiredActions` 只能包含 `approve`、`requestChanges`、`reject`。它表达可用动作，不表达最终决策。

## 合法示例

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Checkpoint
metadata:
  name: idea-to-spec-review-001-direction
spec:
  runRef:
    name: idea-to-spec-review-001
  stage: sharpen-idea
  reason: ConfirmDirectionBeforeSpec
  requiredActions:
    - approve
    - requestChanges
    - reject
  artifactRefs:
    - name: review-notes
status:
  phase: Pending
  conditions:
    - type: CheckpointRequired
      status: "True"
      reason: HumanDesignDecisionRequired
```

对应 source example：`examples/checkpoints/idea-to-spec-review-direction.yaml`。

## 当前 CLI 行为

- `validate` 检查对象形状、动作枚举和 Artifact 引用形状。
- `explain` 输出 `runRef`、stage、reason、required actions、Artifact refs 和通用 conditions。
- 当前 Preflight 不收集独立 Checkpoint 对象，不解析其 `runRef`、stage 或 Artifact refs。
- Lifecycle schema 提供 `checkpoint.requested` 与 `checkpoint.resolved` 事件名，但当前仓库没有 dispatcher。

当前仓库不创建、解决或执行 Checkpoint，也不把某个 status phase 自动同步到 Run。

## 验证边界

- local `runRef` schema 只要求 `name`；它不会强制 `kind: Run` 或目标存在。
- schema 不检查 `stage` 是否属于 Run 引用的 Flow。
- schema 不检查 Artifact 是否来自同一 Run 或 stage。
- schema 不定义 `status.phase` 的 Pending/Approved/Rejected 状态机。
- `expiresAt` 只验证为字符串，不检查过期行为。

## 相邻概念边界

| 容易混淆 | 精确边界 |
| --- | --- |
| Flow checkpoint 声明 | 描述版式中可能在何处停下。 |
| Checkpoint 对象 | 记录某个 Run 的具体决策边界。 |
| Policy `requireApproval` | 声明需要审批的规则；Checkpoint 承载具体决策实例。 |
| Artifact review | 记录产物审查状态；不等于继续执行的决策。 |
| Preflight | 检查机械就绪性；不批准 Checkpoint。 |

## 常见错误

- 把 `requiredActions` 当成已经发生的 action。
- 发明 schema 未定义的 `decision` 或 `resolvedBy` spec 字段。
- 声称当前 CLI 会验证 stage、Run、Artifact 的关系。
- 用 `PreflightReady=True` 替代 Checkpoint 审批。
- 把 Policy 规则内联到 Checkpoint。
- 假设 `expiresAt` 已由 runtime 强制执行。
