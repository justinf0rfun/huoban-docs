`Skill` 是 Huoban 的活字：可复用、可重排的原子能力声明。Skill 描述能力及其边界，不保存项目上下文，不排列 stage，也不执行权限判断。

设计背景见 [对象模型](/object-model)，共享资源外壳见 [规范参考总览](/spec-reference)。

## 字段

| 字段 | 要求 | Schema 语义 |
| --- | --- | --- |
| `spec.capabilities` | 必填，至少 1 项、不可重复 | 每项匹配 `^[a-z0-9.-]+/[A-Za-z][A-Za-z0-9]*$`。 |
| `spec.inputs` | 可选 | 输入数组。 |
| `spec.inputs[].name` | 必填 | 字符串。 |
| `spec.inputs[].type` | 可选 | 字符串；input item 允许其他扩展字段。 |
| `spec.outputs` | 可选 | 输出数组。 |
| `spec.outputs[].name` | 必填 | 字符串。 |
| `spec.outputs[].artifactType` | 可选 | 字符串；output item 允许其他扩展字段。 |
| `spec.profileHints` | 可选 | Profile layer 名称提示的字符串数组，不是对象引用。 |
| `spec.requirements` | 可选 | 外部依赖数组，见 [Requirement](/spec-requirement)。 |
| `spec.sideEffects` | 必填 | 副作用声明对象，不允许其他字段。 |
| `spec.sideEffects.declared` | 必填 | 标准副作用数组，不可重复；可以为空。 |
| `spec.checkpoint` | 可选 | 开放对象；schema 不规定内部字段。 |
| `spec.adapterRef` | 可选 | local object reference，指向关联 Adapter。 |
| `status` | 可选 | 通用状态结构，见 [资源外壳](/spec-reference#资源外壳)。 |

## 副作用枚举

`sideEffects.declared` 只能包含：

```text
readFile, writeFile, deleteFile, executeCommand,
modifyGitIndex, modifyGitHistory, networkAccess, sendMessage,
publishRemote, spendMoney, accessSecret, installDependency,
changePermission
```

这是可能副作用的声明，不是已发生事实，也不是授权。实际观察值可写入 `Run.status.observedSideEffects`；允许、拒绝或审批由 Policy 决定。

## 合法示例

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Skill
metadata:
  name: design-review
spec:
  capabilities:
    - huoban.dev/designReview
  inputs:
    - name: proposal
      type: markdown
  outputs:
    - name: review
      artifactType: decision-notes
  sideEffects:
    declared:
      - readFile
      - writeFile
  checkpoint:
    required: true
    reason: human-design-decision
```

对应 source example：`examples/skills/design-review.yaml`。

## 当前 CLI 行为

- `validate` 检查本页字段、capability 正则、必填项和副作用枚举。
- `explain` 输出 capabilities、Requirement、输入输出、副作用及 checkpoint reason。
- 当 Skill 被 Run stage 或 Lifecycle Handler 绑定时，Preflight 检查它是否提供所需 capability，并检查其 Requirement。
- 如果被绑定 Skill 有 `adapterRef`，Preflight 解析 Adapter，继续检查 Adapter Requirement、`trustRef` 和关联 Policy。
- 未被当前 Run 使用的 Skill 不会因为位于 workspace 就自动接受 Preflight 检查。

当前仓库不执行 Skill。

## 验证边界

- schema 不要求 input/output 名称唯一，也不解释类型兼容性。
- schema 不解释 `profileHints`，不检查 layer 是否存在。
- `checkpoint` 是开放提示结构，不等于一次 [Checkpoint](/spec-checkpoint) 对象。
- 普通 schema validation 不解析 `adapterRef`；当前 Preflight 只在 Skill 可达时解析。
- Skill 的副作用声明不会触发 Policy rule evaluation；当前 Preflight 不执行授权。

## 相邻概念边界

| 容易混淆 | 精确边界 |
| --- | --- |
| Skill / Adapter | Skill 是原子能力对象；Adapter 标准化导入格式或外部能力。 |
| Skill / Profile | Skill 声明能力；Profile 提供分层上下文。 |
| Skill / Flow | Skill 提供 capability；Flow 声明 stage 需要什么 capability。 |
| Skill / Policy | Skill 声明可能副作用；Policy 声明处理规则。 |
| `checkpoint` / Checkpoint | Skill 字段是开放提示；Checkpoint 是绑定具体 Run 的顶层对象。 |

## 常见错误

- 漏写必填的 `sideEffects.declared`；无副作用时应写 `declared: []`。
- 把 capability 写成没有 `/` 的自由文本。
- 把外部环境依赖藏进 input 或 prompt，而不写 `requirements[]`。
- 把 `profileHints` 当作 Profile object reference。
- 认为 Skill 声明了 `writeFile` 就已经获得写权限。
- 把具体执行顺序或 shell 命令写进 Skill；编排属于 Flow，外部执行接口属于 Adapter。
