Huoban（Huó Bǎn）取自“活板”，不是无音调拼音里的“伙伴”（Huǒ Bàn）。Huoban 活板的设计哲学来自活字印刷术。它把 `Skill` 理解为活字，把 `Profile` 理解为墨方，把 `Flow` 理解为版式，把 `Run` 理解为锁版印刷，把 `Checkpoint` 理解为校对，把 `Policy` 理解为印坊规矩。

Huoban 的目标不是先做一个完整 runtime，而是先定义一组可验证、可组合、可审查的标准对象，让 AI 原生工作可以被描述、复用和治理。

## 核心对象

| Object | 隐喻 | 用途 |
| --- | --- | --- |
| `Skill` | 活字 | 可复用、可重排的原子能力。 |
| `Profile` | 墨方 | 分层上下文与项目约束。 |
| `Flow` | 版式 | capability-first 的工作流编排。 |
| `Run` | 锁版印刷 | 一次具体执行的快照。 |
| `Checkpoint` | 校对 | 人类或系统的决策边界。 |
| `Policy` | 印坊规矩 | 副作用、审批和权限规则。 |
| `Adapter` | 转接件 | 标准化现有 skill、MCP tool 或上下文文件。 |
| `Artifact` | 印张 | 工作流产出的文档、报告、代码或记录。 |
| `Registry` | 字架 | 组织可发现、可复用的对象集合。 |
| `Trust` | 验字记录 | 记录来源、审查和信任等级。 |

## 最短验证路径

```bash
npm install
npm test
npm run validate
npm run explain
npm run print:dry-run
```
