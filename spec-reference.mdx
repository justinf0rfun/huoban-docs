Huoban 是 AI 系统的开放编排协议：可排版、可润墨、可印刷。

本节记录 `huoban.dev/v1alpha1` 已由 JSON Schema、examples 和 CLI 实现的协议表面。字段形状以 `schemas/v1alpha1/*.schema.json` 为准；CLI 行为以 `bin/huoban.js` 和 `lib/preflight.js` 为准。

当前 `v1alpha1` 仍是同版本内直接修订的草案，尚无需要兼容的既有用户。不要据此声称旧 reader 能前向兼容后续修订。

## 资源外壳

10 个顶层 kind 都使用同一外壳：

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Skill
metadata:
  name: design-review
spec:
  capabilities:
    - huoban.dev/designReview
  sideEffects:
    declared: []
```

| 字段 | 要求 | Schema 语义 |
| --- | --- | --- |
| `apiVersion` | 必填 | 固定为 `huoban.dev/v1alpha1`。 |
| `kind` | 必填 | 固定为当前对象的 kind。 |
| `metadata` | 必填 | 对象元数据；不允许未声明字段。 |
| `metadata.name` | 必填 | 匹配 `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$`。 |
| `metadata.labels` | 可选 | string 到 string 的映射。 |
| `metadata.annotations` | 可选 | string 到 string 的映射。 |
| `metadata.generation` | 可选 | 大于等于 1 的整数。 |
| `spec` | 必填 | kind 特定的声明内容。 |
| `status` | 可选 | 通用状态，并可由 kind schema 增加字段。 |
| `status.phase` | 可选 | 字符串；通用 schema 不限定枚举。 |
| `status.observedGeneration` | 可选 | 大于等于 1 的整数。 |
| `status.conditions` | 可选 | condition 数组。 |

每个 condition 必须包含 `type`、`status` 和 `reason`。`status` 只能是 `"True"`、`"False"` 或 `Unknown`；`message` 与 `lastTransitionTime` 是可选字符串，schema 不校验时间格式。

## 引用形状

| 结构 | 必填字段 | 可选字段 | 使用位置 |
| --- | --- | --- | --- |
| local object reference | `name` | `kind`、`generation` | `flowRef`、`profileRefs`、`policyRefs`、`runRef` 等。 |
| typed object reference | `kind`、`name` | `generation` | `Trust.spec.subjectRef`、`Requirement.target`、Preflight check 的 `ownerRef`。 |
| artifact reference | `name` | `kind`，若存在只能为 `Artifact` | Checkpoint 和 Run 状态中的产物引用。 |

引用结构只描述形状。普通 `validate` 不解析目标；当前 Preflight 只解析其实现明确收集到的引用。

## Digest 形状

所有协议 digest 使用 `sha256:<64 位小写十六进制>`。对象 digest 使用 Huoban canonical JSON v1：对象 key 按 UTF-16 code unit 升序递归排列、数组顺序保持、按 ECMAScript `JSON.stringify` 语义序列化为 UTF-8，再计算 SHA-256。file input 与 file-backed Artifact 直接哈希原始文件 bytes。

## 10 个顶层 kind

| Kind | 职责 | 规范 |
| --- | --- | --- |
| `Skill` | 声明原子 capability、输入输出、外部依赖与副作用。 | [Skill](/spec-skill) |
| `Profile` | 声明分层上下文及其外部依赖。 | [Profile](/spec-profile) |
| `Flow` | 声明入口输入、capability-first stage、checkpoint 和 lifecycle handler。 | [Flow](/spec-flow) |
| `Run` | 锁定 Flow 引用、调用输入、上下文、策略、实现绑定和快照。 | [Run](/spec-run) |
| `Checkpoint` | 记录一次 Run 的具体决策边界。 | [Checkpoint](/spec-checkpoint) |
| `Policy` | 声明副作用、审批和权限规则。 | [Policy](/spec-policy) |
| `Adapter` | 标准化导入格式或外部能力。 | [Adapter](/spec-adapter) |
| `Artifact` | 保存产物或协议证据的元数据与内容引用。 | [Artifact](/spec-artifact) |
| `Registry` | 声明对象来源与索引范围。 | [Registry](/spec-registry) |
| `Trust` | 记录来源、审查、签名与隔离要求。 | [Trust](/spec-trust) |

## 共享执行契约

以下结构嵌入顶层对象或 Artifact 内容，但不是顶层 kind：

| 结构 | 所属位置 | 规范 |
| --- | --- | --- |
| Flow entry input | `Flow.spec.inputs[]` | [Flow](/spec-flow#顶层字段) |
| Run input binding | `Run.spec.inputBindings` | [Run](/spec-run#input-binding) |
| `Requirement` | `Skill.spec.requirements[]`、`Profile.spec.requirements[]`、`Adapter.spec.requirements[]` | [Requirement](/spec-requirement) |
| Preflight Report | `Artifact.spec.contentRef.data`，且 Artifact type 为 `huoban.dev/preflight-report` | [Preflight](/spec-preflight) |
| Lifecycle Handler | `Flow.spec.lifecycle[]` | [Lifecycle Handler](/spec-lifecycle-handler) |
| Lifecycle Event Envelope | 供 lifecycle 事件使用的共享 fragment | [Lifecycle Handler](/spec-lifecycle-handler#lifecycle-event-envelope) |

把这些结构写成 `kind: Requirement`、`kind: PreflightReport` 或 `kind: LifecycleHandler` 会失败，因为 CLI 找不到对应的顶层 schema。

## 当前验证边界

| 路径 | 已实现 | 未实现 |
| --- | --- | --- |
| `huoban validate` | 解析 YAML/JSON；按 `apiVersion`/`kind` 校验顶层对象，也可校验 Lifecycle Event Envelope fragment；file-backed Artifact 与 Preflight Report 额外验证内容 hash。 | 不解析对象引用，不检查 capability binding，不执行 Policy。 |
| `huoban explain` | 输出对象或 workspace 摘要，显示入口/绑定元数据，并报告一部分缺失引用；inline input 值不输出。 | 不先做 schema validation，也不是完整关系校验。 |
| `huoban print --dry-run` | 验证 `--input`，从 Flow 生成 `mode: dryRun` 的 Run 计划、`inputBindings` 与五类 snapshot hash。 | 不执行 stage，不做 Preflight，不调用外部能力。 |
| `huoban preflight` | 先校验 Run/workspace 对象，再检查可达引用、binding、动态输入内容、Requirement 和当前环境就绪性。 | 不执行 Flow，不调用 MCP tool，不评估 Policy 规则或 Trust 授权。 |
| import/export/derive | Markdown 到 Profile、Profile 到 `AGENTS.md`、Profile risk layer 到 Policy。 | 生成结果不能跳过 review；这些命令不是通用转换 runtime。 |

完整命令和可验证范围见 [兼容与验证](/conformance)。

## 合法示例

上面的最小 `Skill` 是完整、合法的顶层对象。即使没有副作用，也必须保留必填的 `sideEffects.declared: []`。

仓库中的组合示例以 `examples/**/*.yaml` 为准。运行：

```bash
npm run validate
npm run explain
npm run print:dry-run
npm run preflight
```

## 相邻概念边界

| 容易混淆 | 精确边界 |
| --- | --- |
| `spec` / `status` | `spec` 是对象声明；`status` 是工具或执行环境观察到的状态。 |
| schema validation / Preflight | schema 检查对象形状；Preflight 检查锁定 Run 在目标环境中的机械就绪性。 |
| dry-run / Preflight | dry-run 只生成计划；Preflight 对计划做机械检查。二者都不执行 stage。 |
| Preflight / Policy 与 Trust | `PreflightReady=True` 只表示依赖就绪，不表示权限或信任授权。 |
| kind / fragment | kind 有完整资源外壳；fragment 只能嵌入 schema 指定的位置。 |

## 常见错误

- 发明 schema 中不存在的字段，或把开放状态字段误写进 `spec`。
- 把 Requirement、Preflight Report 或 Lifecycle Handler 增加为第 11 个 kind。
- 认为引用形状合法就代表引用目标存在。
- 使用自由字符串 `user.prompt` 或 `stages.a.outputs.b` 代替结构化 `flowInput` / `stageOutput`，或只在 Run 塞入未声明输入。
- 把 stage scope 的 `StagePreflightReady=True` 当成整次 Run 的执行门槛。
- 把 `PreflightReady=True` 当成 Policy allow 或 Trust trusted。
- 声称当前仓库有 Flow executor、Lifecycle dispatcher 或 Policy enforcement runtime。
