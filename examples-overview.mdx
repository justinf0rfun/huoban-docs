## 四条示例路径

| 路径 | 入口 | 可复现结果 |
| --- | --- | --- |
| Profile 迁移 | `examples/import/AGENTS.md` | 生成 Profile、派生待审查 Policy、导出 AGENTS.md。 |
| capability-first 编排 | `examples/flows/idea-to-spec-review.yaml` | 验证 content 入口及其 file carrier，解释 Flow，并打印锁定 refs、bindings 与调用输入的 dry-run Run。 |
| 外部能力适配 | `examples/fixtures/grill-me/SKILL.md` | 验证手写 Adapter、localPath Requirement 和 Trust 记录。 |
| 执行前检查 | `examples/generated/idea-to-spec-review-dry-run.yaml` | 写入 Preflight conditions，并生成 Preflight Report Artifact。 |

这些 `examples/...` 值是 Huoban source repo 内的路径，不是文档站 URL。本章在各示例页内嵌关键完整对象，避免把 source checkout 当成阅读前提。

## 一个完整安全对象

下面逐字取自 `examples/policies/safe-defaults.yaml`：

```yaml
apiVersion: huoban.dev/v1alpha1
kind: Policy
metadata:
  name: safe-defaults
spec:
  defaults:
    unknownSideEffect: requireApproval
  rules:
    - effect: requireApproval
      when:
        sideEffects:
          - publishRemote
          - spendMoney
          - accessSecret
    - effect: deny
      when:
        sideEffects:
          - modifyGitHistory
```

这个对象说明副作用规则，但不会自行执行审批或授予权限。

## 嵌入式执行契约

| 构件 | 所在位置 | 作用 |
| --- | --- | --- |
| Flow entry input | `Flow.spec.inputs[]` | 声明调用者必须提供的动态稿件语义、名称，以及可选 media type / object kind 约束。 |
| Run input binding | `Run.spec.inputBindings` | 保存这次调用的 inline 值、ContentRef 或对象引用与锁定 hash。 |
| `Requirement` | `Skill.spec.requirements`、`Profile.spec.requirements`、`Adapter.spec.requirements` | 声明目标环境依赖及其来源。 |
| Lifecycle Handler | `Flow.spec.lifecycle[]` | 用有限事件、delivery、role、timeout 和 retry 声明观察、上报或清理集成。 |
| Preflight summary | `Run.status.preflight` 与 conditions | 记录 scope、`Run.spec` snapshot hash、检查时间和 report 引用。 |
| Preflight Report | `Artifact` 的 `huoban.dev/preflight-report` 内容 | 分别记录每项 resolution 与 readiness evidence。 |
| Lifecycle Event Envelope | `examples/fragments/stage-succeeded.lifecycle-event.json`、`examples/fragments/checkpoint-requested.lifecycle-event.json` | 给 handler 提供有限、最小、可审计的事件输入。 |
| Lifecycle invocation | `Run.status.lifecycleInvocations[]` | 记录 handler invocation 状态；当前 example 是声明状态，不是 dispatcher 产物。 |

## 复现顺序

在 source repo 根目录运行：

```bash
npm install
npm run validate
npm run explain
npm run print:dry-run
npm run preflight
```

Profile 迁移单独运行：

```bash
npm run import:profile
npm run derive:policy
npm run export:agents
```

完整 deterministic 检查：

```bash
npm test
```

## 不要从示例推断的行为

| 看到的对象状态 | 不能据此声称 |
| --- | --- |
| `mode: dryRun` | 已完成 Preflight。 |
| `PreflightReady=True` | 已通过 Policy、Trust、Checkpoint 或平台授权。 |
| `StagePreflightReady=True` | 整个 Run 已获准执行。 |
| `lifecycleInvocations[].phase: Pending` | 当前仓库存在 Lifecycle dispatcher。 |
| `ImportNeedsReview=True` | 已有自动接受或状态迁移流程。 |

从 [AGENTS.md Profile](/example-import-agents-profile) 开始，或直接进入 [Preflight 与 Lifecycle](/example-preflight-lifecycle)。
