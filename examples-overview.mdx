# 示例总览

示例章节回答一个问题：Huoban 对象如何从真实 AI 工作流中长出来，并被验证、解释、迁移和复盘。

## 复现和验证路径

在 source repo 中运行：

```bash
npm run import:profile
npm run derive:policy
npm run export:agents
npm run print:dry-run
npm run validate
npm run explain
npm run check:generated
```

完整检查可以直接运行：

```bash
npm test
```

预期含义：

| 命令 | 验证什么 |
| --- | --- |
| `npm run import:profile` | 从 `examples/import/AGENTS.md` 重新生成 `examples/generated/agents-profile.yaml`。 |
| `npm run derive:policy` | 从 Profile risk layer 重新生成 `examples/derived/agents-risk-policy.yaml`。 |
| `npm run export:agents` | 从 Profile 重新导出 `examples/export/AGENTS.md`。 |
| `npm run print:dry-run` | 从 Flow 重新生成不执行 agent 工作的 dry-run Run。 |
| `npm run validate` | 所有 YAML 是否符合 `schemas/v1alpha1/`。 |
| `npm run explain` | workspace 是否能被解释成对象族、版式、锁版执行和信任边界。 |
| `npm run check:generated` | 生成文件是否与已提交文件一致，避免 generated/derived/export 产物过期。 |
| `npm test` | 按项目定义串起生成、验证、解释和 deterministic check。 |

`SKILL.md Adapter` 当前没有 import CLI；它验证的是已存在的 Adapter YAML 能被 schema 校验和 explain 解释。

## 对象组合原则

示例遵守三条边界：

- **可排版**：能力、阶段、输入、输出和校对点进入 `Flow`，而不是藏在 prompt 里。
- **可润墨**：项目规则、领域语言和历史决策进入 `Profile`，并可被 `Run.snapshot` 锁定。
- **可印刷**：一次工作通过 `Run`、`Checkpoint` 和 `Artifact` 留下可复盘事实链。

## 常见误读

| 误读 | 正确理解 |
| --- | --- |
| 示例是运行器设计。 | 示例是对象组合设计；runner binding 是下游实现问题。 |
| 导入 Markdown 后就可信。 | 导入对象应带 `ImportNeedsReview`，结构化推断必须可审查。 |
| `Flow` 应绑定具体工具。 | `Flow` 优先声明 capability，`Run.bindings` 再绑定 Skill 或 Adapter。 |
| `Artifact` 是预期输出。 | Flow stage 声明预期输出；`Artifact` 记录一次执行实际留下的产物引用和 review 状态。 |
